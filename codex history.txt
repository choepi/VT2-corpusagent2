You don’t want to start the API outside Docker if your current target is the Docker stack. My last command was wrong for that context.

Use PowerShell to kill only old host processes, then start Docker:

# Kill old host API on 8001
Get-NetTCPConnection -LocalPort 8001 -State Listen -ErrorAction SilentlyContinue |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }

# Kill old host frontend on 5500 if needed
Get-NetTCPConnection -LocalPort 5500 -State Listen -ErrorAction SilentlyContinue |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }

# Verify 8001 is free
Get-NetTCPConnection -LocalPort 8001 -State Listen -ErrorAction SilentlyContinue
Then start the Docker stack from deploy:

cd "D:\Programmieren\GitHub\VT2-corpusagent2\deploy"

$env:CORPUSAGENT2_DOCKER_TORCH_PROFILE="cpu"
